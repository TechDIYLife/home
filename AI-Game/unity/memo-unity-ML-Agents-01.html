1.1 为什么学它？

因为好玩儿

说到AI的应用，通常与好玩儿这件事儿不沾边。 不过如果与游戏结合起来就好玩儿多了。

这么有意思的东西，看着就手痒痒，就入手了一本书来学学 Unity ML-Agents 的游戏制作。

Unity ML-Agents是什么？

  • Unity（游戏制作大众化的工具）： Unity公司开发的游戏开发工具。传统游戏开发是一个非常专业的工作，Unity的目标是游戏制作的大众化，也就是，让普通开发者通过使用Unity可以在几天内制作出高品质的3D游戏。

  • Unity ML-Agents（AI游戏大众化）：Unity工具中用于构建强化学习开发环境的开发库。 通过此工具我们可以来训练Agent来完成各种任务，比如自主玩游戏，走迷宫，驾驶汽车等。对游戏开发者来说，还可以用它来实现游戏测试的自动化，以及对游戏设计进行评价等等。

1.2 基本概念

机器学习的基本术语
以下机器学习词汇，都属于基本的机器学习内容，不清楚的话可以去问chatGPT。

  • 机器学习
  • 深度学习
  • 神经元，神经网络
  • 模型设计，模型训练，模型推理
  • 监督学习，非监督学习，强化学习，模仿学习
  • 分类，回归，聚类

强化学习

这里的强化学习，学习的是一种行动策略。这种行动策略用来帮助一个智能体根据所处环境中的状态情况，来选择行动，比如向左，向右还是跳一下。

当智能体，做得好就给于奖励，做的不好进行惩罚，这种让智能体学习的方式就是强化学习。

--- title: 图1.2.1 行动策略模型 --- graph LR A[输入：环境的状态] --> B[行动策略模型] --> C[输出：行动]

强化学习相关的术语

  • 智能体（Agent）：在Unity ML-Agents的环境中，一个智能体可能是一个游戏中的角色、一个机器人。这些智能体通过观察它们所处的环境（通常是以数值数据的形式来表示当前的状态），然后根据观察的结果来做出决策（即选择动作）。动作的结果会影响环境，从而为智能体提供奖励或惩罚的反馈。AI算法则根据这些反馈来调整其未来的行动策略。

  • 环境（Environment）：智能体所处的世界。

  • 行动（Action），状态（State）：行动就是可以进行的动作，比如前进，后退等。智能体根据当前的状态来预测行动，执行行动之后，智能体所处的状态就会跟着改变。

  • 报酬（Reward）：是指对智能体行动的惩罚或者奖励。报酬不仅与【行动】有关还与【状态】有关，比如，向右边走，可能是吃到蘑菇加分，也可能是碰到怪物，被杀死。吃到蘑菇，则给于【奖励】，被杀死就给于【惩罚】。

  • 及时报酬，延迟报酬：及时报酬，行动后立即得到的报酬； 行动后，过了一段时间产生的报酬就叫延迟报酬。延迟报酬是一种高级策略性的选择，出发的动机是短期回报，还是长期回报。比如短期看起来有益的东西，可能从长期来看对最终目标是有害的。

  • 策略（Policy）： 强化学习是根据现在的状态来决定下一步的行动。这种确定下一步行动的战略就被称为策略。 强化学习的目的就是要得到报酬最大化的策略，也就是训练后的模型。

  • 收益（return）和价值（Value）：收益是及时报酬，以及其后所有报酬的总和。 强化学习学习的目标为收益最大化。所以，报酬是每一次行动给与的奖励或者惩罚，而收益则是此次行动导致的所有报酬的总和。因此，收益不仅包含及时报酬，还包含延迟报酬。
    但是，由于未来的事情还没有发生，并且会受到未来的各种变量的影响，所以收益难以确定。为了计算未来的收益，将状态和策略来固定来计算收益，此时的收益被成为价值。

    所以，如下图所示，优化的方式是通过优化【价值最大化】来近似的优化【收益最大化】，从而获得【报酬最大化】的策略。

    --- title: 图1.2.2 强化学习训练时的近似优化 --- graph LR; A[价值最大化] --> B[收益最大化] --> C[获得报酬最大化的策略]

强化学习相关术语的总结：

用语 说明 无人岛生存的例子
智能体 在环境中活动的主体 求生的人
环境 智能体所处的世界 无人岛
行动 智能体在所处的环境中可以采取的行动 移动， 休息， 饮水，采集食物等等
状态 确定智能体当前所处的环境的所有因素 人的位置，身体状况，天气状态等等
报酬 环境对智能体采取行动的评价 行动导致生存概率提高，则给与奖励
策略 智能体进行行动选择的机制，原理 人的生存方法
及时报酬 采取行动后马上可以得到的报酬 休息后体力回复
延迟报酬 行动后得到的结果 经过探索，发现了食物（探索本身没有报酬，但是获得食物之后得到了报酬）
收益 及时报酬和延迟报酬的总和 -
价值 智能体在状态和策略固定的情况下可以获得收益 -

1.3 强化学习流程图

--- title: 图1.3.1 强化学习流程 --- graph LR; A[ 环境 ] --1 状态获取\n4 报酬获取--> B[智能体] -- 5 根据报酬来更新策略 --> C[策略模型] B -- 3 行动执行--> A C --2 根据状态等来选择行动--> B

学习的流程：

  1. 【智能体】通过观察【环境】来取得当前【状态】
  2. 使用现在的【策略】根据取得的【状态】决定下一步的【行动】。最开始的【策略】可能是随机的，但是随着学习的进行，策略会越来越好。
  3. 执行选择的【行动】
  4. 【行动】执行后，会对【环境】产生影响，从而获取环境反馈的【报酬】（什么样的影响，给什么报酬是需要我们来定义的）
  5. 根据【报酬】来优化【策略】，优化的目标是让【报酬】最大化。

上面的1到5步是一个学习的Step，整个过程反复的迭代循环，最终得到一个【报酬】最大化的【策略模型】。
这个训练的过程被成为马尔可夫决策过程。 马尔可夫决策过程是指，下一个状态，是由现在的状态和行动决定的，

学习流程相关的几个概念

  • 回合，关卡（episode）：强化学习的episode类似于一局游戏，一个回合。一局游戏，即游戏开始到游戏结束，即为一个episode。强化学习优化的目标是一个回合中获得报酬总和的最大化。
  • 步（Step）：一次行动被称为一步。也就是游戏的一次响应，或者成为一帧（frame）。收益计算的基本单位就是这个Step。
  • 探索，应用： 假设无人岛上，食物在毒蛇的附近，如果因为害怕毒蛇，而远离毒蛇，那么结果是找不到食物而饿死。因此靠近毒蛇表面看起来是不好的行为，但是对于找到食物来说，是正确的行为。为了整体收益最大，可能就需要更大胆一些。
    因此探索，就是要一定程度上忽视固有的经验采取更大胆的行动，而应用就是更多的利用以前学习的经验。通过调整学习率可以来控制更倾向于探索还是应用。当使用较大的学习率时，更倾向于探索，学习率小，则是更倾向于应用。

1.4 利用 Unity ML-Agents 强化学习

Unity ML-Agents 是 Unity Machine learning Agents Toolkit 的简写，他是一个 Unity 的开发库。通过此开发库，你可以在Unity环境中使用强化学习来训练智能体。

训练后的智能体，可以控制游戏中的角色玩游戏，对游戏开发者来说，还可以用它来实现游戏测试的自动化，以及对游戏设计进行评价等等。

Unity ML-Agents 三个主要部件

  • 环境：Unity 场景以及其中包含的所有的游戏角色，元素等。
  • 智能体：Unity 环境内进行学习和推论的游戏角色。
  • 模型训练用Python脚本：用于训练智能体的 Python AI 程序，这些脚本在终端中执行，它通过与 Unity 环境进行通信来训练智能体。

三种模式：训练模式，推理模式，人类玩家模式

--- title: 图1.4.1 学习模式 --- graph RL subgraph sub1[Unity环境] direction RL A[环境] --1 状态获取\n4 报酬获取--> B[智能体] B -- 3 行动执行--> A end subgraph sub2[Python脚本] direction TB B -- 5 根据报酬情况来更新策略 --> C[策略模型] C --2 根据状态等来选择行动--> B end

学习模式下，Unity环境复杂管理游戏环境，智能体与环境的交互， Python脚本部分通过与Unity环境，执行行动，获取智能体状态，并通过其AI程序来对策略模型进行优化。

--- title: 图1.4.2 推理模式 --- graph LR subgraph Unity环境 direction LR A[环境] B[智能体 \n 策略模型【决定行动】] A --1 状态获取--> B B -- 3 行动执行--> A end

推理模式下，由于不需要对模型进行更新，所以并不需要Python脚本部分，所以可以将优化好的【策略模型】与游戏一起发布。

--- title: 图1.4.3 人类玩家模式 --- graph RL A[环境] B[游戏角色] C[人类玩家] subgraph Unity环境 direction TB A --1 状态获取--> B B -- 3 行动执行--> A end subgraph 玩家 direction TB B -- 画面 --> C C --行动--> B end

人类玩家模式，与学习模式是非常相似的。玩家通过画面来获取环境的状态，并做出判断来控制游戏角色来完成各种任务。

相关概念

  • 模仿学习：智能体通过模仿人类的操作来学习的一种学习方式。

  • 自我博弈（self-play）：一种通过自我对抗进行学习的学习方式。著名的Alpha-Zero就是通过自我博弈的方式训练出来的。

  • 好奇心（Curiosity）：促进智能体对未知状态进行探索的学习方法。

  • 课程学习（Curriculum Learning） （先易后难学习）：参考学生上学学习的方式，先学习简单的概念或技能，逐渐过渡到学习复杂的问题。

  • 环境参数随机化：对环境进行初始化设置？。

  • LSTM（Long Short-Term Memory）：长短时记忆神经网络，可以利用过去的状态来做预测的神经网络结构。

2.2 安装步骤

安装Unity Hub和Unity
Unity Hub是一个 Unity 的安装和管理的工具，它支持多个Unity版本，License管理，项目管理等等。

安装步骤如下：

  1. Unity Hub 的安装（下载地址：https://unity.com/download ）
  2. Unity 账号创建（需要邮件验证）
  3. Unity Hub 端用户登录（点击Sign in登录）【如果使用网络代理，请参考常见问题Q1】
  4. 添加 License： 左上用户头像->Manage licenses->Add(选择：Get a free personal license)
  5. 安装 Unity 2023.1.20f1（Window环境选择：Windows Build Support模块）

注意：
如果你使用网络代理，会遇到很多的问题。

  1. Unity Hub用户登录问题

2.3 常见问题

  • Q1：Unity Hub login (Something went wrong. Please sign in again.)
    原因分析：无法访问网络造成的。你的网络环境可能需要设置代理才可以访问网络。

    • Q1A1: 在Windows环境中配置HTTP_PROXY, HTTPS_PROXY, NO_PORXY （）

    • Q1A2：官方介绍的脚本启动方法（在我的环境中不work）：
      新建要给文件，比如 UnityHubProxy.cmd
      添加以下内容：

      @echo off
set HTTP_PROXY=http://<proxy-address>:<proxy-port>
set HTTPS_PROXY=http://<proxy-address>:<proxy-port>
start "" "C:\Program Files\Unity Hub\Unity Hub.exe"

      保存文件并以管理员身份运行脚本
      https://docs.unity3d.com/ja/2023.2/Manual/ent-proxy-cmd-file.html

    • Q1A3：类似脚本启动方法的手动登录
      Q1A2的方法，在账号登录时出现问题（浏览器端登录成功后，无法重新链接并激活Unity Hub程序完成账号登录）
      手动登录方法如下：

      1. 以管理员身份启动一个终端
      2. 设置代理，启动Unity Hub
      set HTTP_PROXY=http://<proxy-address>:<proxy-port>
set HTTPS_PROXY=http://<proxy-address>:<proxy-port>
