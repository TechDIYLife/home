介绍 | 🦜️🔗 Langchain

LangChain 是一个用于开发由语言模型驱动的应用程序的框架。它使应用程序能够：

  • 具有上下文感知能力：将语言模型与上下文源连接（提示指令、少量示例、用于支撑其响应的内容等）。
  • 进行推理：依靠语言模型进行推理（基于提供的上下文如何回答、采取何种行动等）。

这个框架包含几个部分。

  • LangChain 库：Python 和 JavaScript 库。包含各种组件的接口和集成，一个基本的运行时，用于将这些组件组合成链和代理，以及现成的链和代理的实现。
  • LangChain 模板：一系列易于部署的参考架构，适用于各种任务。
  • LangServe：一个用于将 LangChain 链部署为 REST API 的库。
  • LangSmith：一个开发者平台，让你可以调试、测试、评估和监控建立在任何 LLM 框架上的链，并且与 LangChain 无缝集成。

这些产品共同简化了整个应用程序生命周期：

  • 开发：用 LangChain/LangChain.js 编写应用程序。使用模板作为参考快速启动。
  • 生产化：使用 LangSmith 检查、测试和监控你的链，以便你可以不断改进并充满信心地部署。
  • 部署：将任何链转换为 API，使用 LangServe。

LangChain Libraries

LangChain 包的主要价值主张是：

  1. 组件：用于处理语言模型的可组合工具和集成。无论你是否使用 LangChain 框架的其他部分，组件都是模块化的、易于使用的。
  2. 现成的链：内置的组件组合，用于完成更高级的任务。

现成的链使开始变得简单。组件使自定义现有链和构建新链变得简单。

LangChain 库本身由几个不同的包组成。

  • langchain-core：基本抽象和 LangChain 表达语言。
  • langchain-community：第三方集成。
  • langchain：构成应用认知架构的链、代理和检索策略。

LangChain 表达语言 (LCEL)

LCEL 是一种声明式的方式来组合链。LCEL 从第一天开始就被设计为支持将原型投入生产，无需代码更改，从最简单的“提示 + LLM”链到最复杂的链。
