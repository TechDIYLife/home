Transformers开发：使用 auto_map 开发自定义模型

1 内容概要

Transformers开发库中，支持260多种类型的模型。这些模型的处理实现代码，可以在 src/transformers/model 目录下找到。

如果，要开发自定义的模型，该如何让transformers也支持我们的模型呢？

解决方法就是，通过使用 transformers 提供的 auto_map 的功能。
通过这个功能，可能让我们非常方便的制作甚至分享自定义模型。

2 通过 auto_map 开发自定义模型

2.1 对比一下LLaMA和Qwen模型

Qwen模型（第一版不被Transformers支持，类似于自定义模型）：千问通义第一版的公开的模型中，不仅有模型参数，还包括几个python文件。这几个python文件，与LLaMA的实现处理文件的作用是类似的。

2.2 Auto_map的使用方法

要实现千问模型的自定义模型开发，需要准备以下代码：

  1. 准备模型实现的源代码： configuration_qwen.py, modeling_qwen.py, tokenizaiton_qwen.py

  2. 设置配置文件，让 transformers 调用自定义代码：

设置方式如下：
config.json

"auto_map": {
    "AutoConfig": "configuration_qwen.QWenConfig",
    "AutoModelForCausalLM": "modeling_qwen.QWenLMHeadModel"
  },

tokenizer_config.json

  "auto_map": {
    "AutoTokenizer": [
      "tokenization_qwen.QWenTokenizer",
      null
    ]
  },
  1. 将python代码与模型放到同一目录下发布。

2.3 举例：基于LLaMA2进行自定义模型开发

如果基于LLaMA2模型进行自定义模型的开发，操作步骤参考如下：

  1. 将 configuration_llama.py, modeling_llama.py, tokenizaiton_llama.py 拷贝到模型所在目录下。

  2. 在配置文件中，添加下面内容：

config.json

  "auto_map": {
    "AutoConfig": "configuration_llama.LlamaConfig",
    "AutoModelForCausalLM": "modeling_llama.LlamaForCausalLM"
  },

tokenizer_config.json

  "auto_map": {
    "AutoTokenizer": [
      "tokenization_llama.LlamaTokenizer",
      null
    ]
  },

3 总结

auto_map功能是transformers提供的用于开发自定义模型的一个功能，非常的方便，实用， 通过这个功能进行自定义的模型开发，有以下优点：

  • 方便共享：自定义的模型可以在标准的 transformers 下运行，不需要对transformers代码进行修改。
  • 易于管理：模型与代码是对应的，版本管理更加方便。
