Linux上安装 Ollama + Open WebUI

整理的我在 Linux 上安装 Ollama 和 Open WebUI 是做的笔记记录。
其中包括 Ollama 的安装， Docker安装，以及通过docker 安装Open WebUI。

1 安装Ollama

Q1.1：如何以 root 权限安装 ollama？

有root权限的用户可以通过执行下面的命令安装：

curl -fsSL https://ollama.com/install.sh | sh

注意：这个脚本会做什么：

  • 第一步 文件下载：根据您的Linux版本，自动下载对应的 ollama 二进制执行文件，并默认存放在下面目录下 /usr/local/bin (备选：/usr/bin； /bin)
  • 第二步 新建用户：新建一个 ollama 的用户
  • 第三步 注册服务：创建服用用配置文件（/etc/systemd/system/ollama.service），并启动 ollama.service 的服务。
    查看服务的命令：
    # 列出所有服务（包括活动的和非活动的）:
systemctl list-unit-files --type=service
# 只列出正在运行的服务:
systemctl --type=service --state=running
# 检查特定服务的状态:
systemctl status <服务名称>

# 服务相关命令：
启动服务：sudo systemctl start ollama.service
停止服务：sudo systemctl stop ollama.service
重启服务：sudo systemctl restart ollama.service
查看服务状态：sudo systemctl status ollama.service
设置服务开机自启：sudo systemctl enable ollama.service
  • 第四步 检查CUDA：会自动检查并安装CUDA驱动

Q1.2：如何以Linux普通用户（无root权限）安装？ 

# 下载
curl -L https://ollama.com/download/ollama-linux-amd64 -o /home/xxxx/bin/ollama
# 修改文件属性
chmod +x /home/xxxx/bin/ollama

# 将路径加入用户PATH中(推荐)
export PATH=/home/xxxx/bin:$PATH

# 启动
ollama serve

无root权限无法设置作为服务启动。
手动加入服务方法，请参考：https://github.com/ollama/ollama/blob/main/docs/linux.md

无启动服务的影响：
个人使用影响不大。
如果要使用ollama提供服务，建议设置为系统服务。

Q2.1：如何安装Docker？

如果没有安装Docker，可以参考下面的命令进行安装：

sudo apt-get update
sudo apt-get install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings

sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
# 如果需要代理，可以参考执行下面命令，将xxxxx替换为你的代理服务器
sudo https_proxy="xxxxxxx" curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc

sudo chmod a+r /etc/apt/keyrings/docker.asc

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 测试Docker是否可以使用
sudo docker run hello-world
sudo docker ps

Q2.2：如何为Docker设置网络代理 http_proxy？

sudo mkdir /etc/systemd/system/docker.service.d
sudo nano /etc/systemd/system/docker.service.d/http-proxy.conf
文件内容如下：

[Service]
Environment="HTTP_PROXY=http://xxxxxxxx"
Environment="HTTPS_PROXY=http://xxxxxxxx"
Environment="NO_PROXY=127.0.0.1,localhost,192.168.0.0/16"

重启服务：

sudo systemctl daemon-reload
sudo systemctl restart docker

Q2.3：如何下载安装Docker版Open WebUI？

下面两种方式，都可以启动成功。

# WebUI与ollama在同一台机器：
sudo docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main

# WebUI与ollama不在同一台机器：
docker run -d -p 3000:8080 -e OLLAMA_BASE_URL=https://example.com -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main

# 访问方法: 
http://localhost:3000

如果上面命令无法正确启动（建议安装后等待30分钟后，再确认），可以执行下面命令：

sudo docker run -d --network=host -v open-webui:/app/backend/data -e OLLAMA_BASE_URL=http://127.0.0.1:11434 --name open-webui --restart always ghcr.io/open-webui/open-webui:main

# 访问方法
http://localhost:8080

相关的原因及解释，请参考： https://docs.openwebui.com/getting-started/

Q2.4：安装后通过浏览器访问不出现登录界面，该如何处理？

  • 建议一：安装后，无法访问，建议等待十几分钟后再进行确认（原因不明）比如：我的自己的情况是15分钟后，访问就ok了。
  • 建议二：使用下面的命令进行安装
sudo docker run -d --network=host -v open-webui:/app/backend/data -e OLLAMA_BASE_URL=http://127.0.0.1:11434 --name open-webui --restart always ghcr.io/open-webui/open-webui:main
  • 建议三：检查服务是否启动成功
# 确认ollama是否在运行：
命令：curl http://127.0.0.1:11434
返回：Ollama is running

# 确认webui是否正常运行
curl localhost:8080
curl localhost:3000

# 登录docker container中检查服务是否启动
sudo docker exec -it <CONTAINER ID> /bin/bash
  #设置http_proxy, https_proxy，no_proxy
  apt update
  apt-get install curl
  apt-get install nano

Q2.5：如何为Linux系统设置网络代理服务器？

在Linux环境中，设置方式如下：

export http_proxy=http://xxxxxx
export https_proxy=http://xxxxx
export no_proxy=127.0.0.1,localhost,192.168.0.0/16

Q2.6：常用的 Docker 管理命令有那些？ 

# 列出运行的container
docker container ls

# 列出所有的container
docker container ls -a

sudo docker logs <containerid> #日志
sudo docker start <containerid> #启动
sudo docker stop <containerid> #停止
sudo docker restart <containerid> #重启

# 如何删除container
docker ps -a
docker rm <containerid>
docker rm -f <containerid> #强制删除在运行的container

# 如何删除images
docker images
docker rmi <imageid>

3 通过源代码安装 Open WebUI

安装步骤：

  • 安装Node.js
  • 准备Python运行环境
  • 安装Open WebUI

Q3.1：如何安装node.js ？ 

sudo apt update
sudo apt-get install -y nodejs

如果安装时出现错误：npm : Depends: node-gyp (>= 3.6.2~) but it is not going to be installed
请按下面方式进行安装

sudo apt remove --purge nodejs npm
sudo apt clean
sudo apt autoclean
sudo apt install -f
sudo apt autoremove
sudo apt install curl
cd ~
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
curl -sS https://dl.yarnpkg.com/debian/pubkey.gpg | sudo apt-key add -
echo "deb https://dl.yarnpkg.com/debian/ stable main" | sudo tee /etc/apt/sources.list.d/yarn.list
sudo apt update
sudo apt install yarn
node -v && npm -v

https://stackoverflow.com/questions/55464934/npm-depends-node-gyp-3-6-2-but-it-is-not-going-to-be-installed

Q3.2：如何通过源码安装 open webui？

设置python运行环境

conda create -n webui python=3.10
conda activate webui

下载源码进行安装

git clone https://github.com/open-webui/open-webui.git
cd open-webui/

# Copying required .env file
cp -RPp .env.example .env

# Building Frontend Using Node
npm i
npm run build

# Serving Frontend with the Backend
cd ./backend
pip install -r requirements.txt -U
bash start.sh

官方文档：https://docs.openwebui.com/getting-started/
