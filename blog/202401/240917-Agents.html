1.1 Assistants API 是什么？

Assistants API 允许你在自己的应用程序中构建 AI 助手。一个助手包含指令，可以利用模型、工具和附件文档来回答用户的提问。
Assistants API 目前支持三种类型的工具： 代码解释器 Code Interpreter， 文件搜索 File Search， 自定义函数调用 Function calling

1.2 Assistants API 都有哪些功能？

首先，看一下Assistants 都有哪些功能。Assistants API 目前依然是一个beta版本，当前提供的功能如下：

  1. 模型个性化设置：助手可以调用 OpenAI 的模型，通过特定的指令来调整它们的个性和能力。（GPTs 就类似一个助手）
  2. 支持多个工具，并行访问：助手可以并行访问多个工具，这些工具可以是由 OpenAI 托管的工具（如 code_interpreter 和 file_search），也可以是你自己构建/托管的工具（通过函数调用）。
  3. 支持历史对话记录：助手可以访问持久化的对话线程。线程通过存储消息历史记录并在对话超过模型的上下文长度时进行截断，简化了 AI 应用程序的开发。你只需创建一次线程，并在用户回复时将消息附加到该线程中。
  4. 支持访问文件：助手可以访问多种格式的文件——无论是在助手创建过程中还是在助手与用户之间的对话线程中。当使用工具时，助手还可以创建文件（例如图片、电子表格等），并在它们创建的消息中引用这些文件。

1.3 Assistants 处理过程是什么样的？

Assistant：使用OpenAI模型并调用工具的专门构建的AI
Thread：助理与用户之间的会话。线程存储消息，并自动处理截断以适应模型的上下文。
Message：由助理或用户创建的消息。消息可以包括文本、图像和其他文件。消息作为列表存储在线程上。
Run：在线程上调用助理的行为。助理使用其配置和线程的消息执行任务，通过调用模型和工具。作为运行的一部分，助理将消息附加到线程上。
Run Step：助理在运行中采取的详细步骤列表。助理可以在其运行期间调用工具或创建消息。检查运行步骤可以让您反思助理如何得出最终结果。
