大家都在用哪个LLM本机推理，部署工具？

1 介绍

随着开源大型语言模型（LLM）的不断增多，本地运行和部署这些模型的工具也相应涌现出来。如同大型模型本身，每种工具都宣称自己具有独特的优势。面对这样一个多样化的选择，用户应该如何决定哪种部署工具最适合自己的需求呢？

今天，我将从两个关键维度来审视这些推理和部署工具，以期帮助大家筛选出最符合自己需求的工具。这两个维度是：被大型模型推荐的情况，GitHub上的统计数据。

  • 被大型模型推荐的情况：这一维度关注的是各个工具是否得到了领先大型模型开发者或组织的推荐。这样的推荐通常意味着工具在性能、兼容性和易用性方面已经得到了一定程度的验证，能够更好地支持特定的模型和任务。

  • GitHub上的统计数据：通过观察GitHub上的相关统计数据（如星标数量、贡献者数量、fork次数等），我们可以从社区活跃度和用户接受度两个角度来评估工具的普及程度和稳定性。一个活跃的社区通常意味着工具能够快速迭代和修复，以及有丰富的使用案例和解决方案可供参考。

结合这两个维度的分析，可以为选择最适合自己需求的部署工具提供一个更全面的视角。

推理，部署工具分类：

  • 本机推理（local inference）
  • 部署（Deployment）
  • WebUI
  • 开发平台
  • 其他：云服务相关，独特用途等等

2 八大开源大模型推荐了哪些工具？

统计方法： 主要参考以下8个开源模型的推荐部署方法，这些模型分别是：
Mistral，LLaMA，Falcon，Gemma，ChatGLM，Qwen，Yi，Baichuan。

推荐阅读：《开源大模型下载量排行榜（2024年3月更新版）》
https://techdiylife.github.io/blog/blog.html?category1=c01&blogid=0030

2.0 HuggingFace Transformers

  • LLM界的大哥大，最主要推动者
  • 特点：最基本的使用方法，当前绝大多数LLM都基于此开发库开发。支持Window，Mac，Linux，支持Web？
  • 推荐的模型：全部

2.1 Local inference 本机推理工具

  • llama.cpp： https://github.com/ggerganov/llama.cpp

    • 特点： 基于C/C++，支持Window，Mac，Linux； 支持AMD显卡；API访问支持（兼容OpenAI API），支持Multimodal models
    • 推荐的模型：llama支持，qwen1.5，Yi
    • 使用方法
      Yi模型提供： https://github.com/01-ai/Yi?tab=readme-ov-file#quick-start---llamacpp

  • ollama： https://github.com/ollama/ollama

    • 官网：https://ollama.ai/
    • 特点：llama官方文档称其为最简单的方式, 图标非常可爱。支持Window，Mac，Linux； 支持AMD显卡；支持视觉模型；REST API
    • 推荐的模型：llama，qwen1.5

  • HuggingFace TGI： https://github.com/huggingface/text-generation-inference

    • 特点：Huggingface出品，支持Window，Mac，Linux； 支持AMD显卡；API
    • 推荐的模型：llama
    • 使用方法
      llama模型提供：https://github.com/meta-llama/llama-recipes/tree/main/recipes/inference/model_servers/hf_text_generation_inference

  • TensorRT-LLM （NVidia）：https://github.com/NVIDIA/TensorRT-LLM

    • 特点：英伟达出品，2023年10月开始
    • 推荐的模型：mistral

  • MLC LLM： https://github.com/mlc-ai/mlc-llm

    • 特点：目标是要支持所有平台，ios，Android，Web Browser，也支持Linux，Win，macOS
    • 推荐的模型：llama 推荐ios，android量化版运行工具

  • liteLLM：https://github.com/BerriAI/litellm

    • 特点： 通过OpenAI API格式方法LLM
    • 推荐的模型：llama推荐的支持OpenAI Style API

  • LlamaIndex：https://github.com/run-llama/llama_index

    • 特点：支持RAG应用（retrieval augmented generation ）
    • 推荐的模型：Qwen模型提到

  • 闻达：https://github.com/wenda-LLM/wenda

    • 特点：国产，支持知识库对接，本地，在线搜索。对话管理，多用户支持
    • 推荐的模型：ChatGLM有提到

  • llamafile：https://github.com/Mozilla-Ocho/llamafile

    • 特点：基于llama.cpp开发的工具
    • 推荐的模型：qwen1.5提到

  • ScaleLLM：https://github.com/vectorch-ai/ScaleLLM

    • 特点：小Team，开发中
    • 推荐的模型：Yi

2.2 Deployment 部署工具

  • vLLM： https://github.com/vllm-project/vllm

    • 特点： Qwen将其分类为部署工具
    • 推荐的模型：qwen1.5； mistral， llama2
    • 使用方法：
      llama2使用实例：https://github.com/meta-llama/llama-recipes/tree/main/recipes/inference/model_servers

  • SGLang： https://github.com/sgl-project/sglang

    • 特点： 侧重部署
    • 推荐的模型：qwen1.5提到

2.3 应用开发

  • LangChain：https://github.com/langchain-ai/langchain
    • 特点：基于LM开发应用的framework
    • 推荐的模型：llama

2.4 WebUI

  • oobabooga： https://github.com/oobabooga/text-generation-webui
    • 推荐的模型：qwen1.5

2.5 其他工具

  • LM Studio：https://lmstudio.ai/

    • 特点：本地模型运行集成工具，安装包方式提供。基于llamp.cpp开发
    • 推荐的模型：qwen1.5提到

  • SkyPilot：https://github.com/skypilot-org/skypilot

    • 特点：Run LLMs and AI on Any Cloud （在云服务上部署后，可以通过API来访问。）
    • 推荐的模型：mistral

  • continue：https://github.com/continuedev/continue

    • 网站：https://continue.dev/
    • 特点：支持多种模型的代码生成辅助工具（支持vscode）
    • 推荐的模型：llama提到

  • llm-vscode：https://github.com/huggingface/llm-vscode

    • 特点：支持vscode的扩展库
    • 推荐的模型：llama推荐

  • fastllm：https://github.com/ztxz16/fastllm

    • 特点：fastllm是纯c++实现，无第三方依赖的高性能大模型推理库
    • 推荐的模型：chatglm

  • KerasNLP：https://github.com/keras-team/keras-nlp

    • 推荐的模型：Gemma

  • Vertex AI：谷歌云平台

    • 推荐的模型：Gemma
