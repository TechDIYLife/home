2 RAG评估指标：两种视角理解评估指标

可以通过下面两个视角来梳理RAG评估指标，LangChain的模式Mode视角，RAGAS的功能模块Component-wise视角

  • LangChain评估方法（Mode视角）

    • 评估全景图（来自LangChain官方资料）

      与RAG评估相关的几点解释：
      a. 数据集Dataset的制作方法：Manually curated（手工制作），Form user logs（从执行日志中提取）， Synthetic（使用LLM生成）
      b. 评估器Evaluator的评估方法：LLM-as-judge（LLM评估打分），Heuristics（基于规则的判断），Human（人工评价）
      c. 评估模式Mode：Comparison（比较生成对），Reference-free（没有正确答案时，通过一个标准来比较）， Ground Truch（将解答与正确答案比较）

      参考资料：
      https://docs.google.com/presentation/d/1y_dYspY8odgVvE4OfimCLmw9QHz7QPkYNMvK8wyg9bY/edit?usp=sharing
      https://docs.google.com/presentation/d/1N1JKe_7qLIuiY2M40gKFGGYTb37t6F_oqGLUpw5oXy4/edit?usp=sharing

    • 通过Mode视角，可以将[教程1] 中评估指标整理如下：

      （1）Response <> reference answer：正确性（answer correctness） 用于“解答与真实标签（ground-truth label）相比有多相似/正确”
      （2）Response <> input（question）：答案相关性（answer relevance）、有用性（answer helpfulness） 等用于衡量“生成的回答在多大程度上解决了用户的初始问题”
      （3）Response <> retrieved docs：真实度（faithfulness）、幻觉（answer hallucination） 等用于衡量“生成的回答与检索到的上下文在多大程度上一致”
      （4）Retrieved docs <> input（question）：上下文相关性 (document_relevance) 等用于衡量“检索到的结果与查询有多相关”。

    • 通过Mode视角，可以将[教程2]中列出的RAGAS对RAG的评估指标整理如下：

      （1）Response <> reference answer：答案正确性 (answer_correctness) 根据真实答案 (ground-truth label) 来判断响应是否是正确的。
      （3）Response <> retrieved docs： 可信度 (faithfulness) 评价产生的响应中有多少是基于检索到的上下文。
      （4）input（question） <> retrieved docs：上下文相关性 (context_relevancy) 检索到的句子中有多少与用户问题是“相关”的。
      （5）retrieved docs <> reference answer：上下文召回率 (context_recall) 衡量检索到的上下文与真实答案的匹配程度。
      （6）retrieved docs <> input（question），reference answer：上下文准确率 (context_precision) 用于判断相关文档在上下文中的排名是否靠前。（计算方法较复杂，可以参考此官方文档中的Calculation）

  • RAGAS评估方法（功能模块视角）

    • Component-Wise Evaluation

      • Faithfulness 解释见前面
      • Answer relevancy：（Input <> Response）根据response解答来生成问题，比较生成问题与原问题的相似度。
      • Context recall 解释见前面
      • Context precision 解释见前面
      • Context relevancy 解释见前面
      • Context entity recall（检索文本召回率）：（Retrieved docs <> Reference docs），用来评估检索文本的召回率，需要相关文本的真实答案。

    • End-to-End Evaluation

      • Answer semantic similarity ，（Response <> reference answer）通过计算response解答和真实答案的向量的相似度来评价解答与答案有多相似。
      • Answer correctness 解释见前面

      更多详细内容请参考：RAGAS Metrics

注1：评估时，不需要使用所有这些指标，每个指标都能在不同方面为检索器和生成器提供参考。
注2：以上评估方法都依赖于大语言模型，默认选择OpenAI的GPT4。

3 快速上手指南

RAG评估指标很多也很复杂，开始上手时建议先选择两个指标。快速上手流程如下：

  1. 选择目标数据，准备测试数据集（可以只有问题）
  2. 准备基本的RAG程序
  3. 根据数据集选择两个指标：
    第一个偏重生成（比如正确性correctness）
    第二个偏重检索（比如上下文相关性，上下文召回率，HitRate/MRR）
  4. 根据评估结果迭代
