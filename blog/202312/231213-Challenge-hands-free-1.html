1 语音对话部分：识别，对话，合成

1.1 问题描述

目标是什么？
实现Hands free的语音对话功能：

  1. 收录：收集语音，存储为wav文件 （原因：API只接受语音文件）。
  2. 识别：使用API对wav文件进行识别。
  3. 对话：将识别结果，提交给ChatGPT
  4. 合成：结果返回，使用语音合成功能合成语音
  5. 播放：播放语音

挑战分析？

  • 第一，录音问题：
    只收录说话的声音，没有说话时，可以自动停止录音 --> 可以通过声音能量来判断
    挑战1：一句话中间出现暂时中断时，可能会造成录音中断的问题。
    挑战2：当说错了，重新说时，是否会影响模型的理解？
    --> 难道需要我像个电视台的播音员一样，吐字清晰，说话连续，稳定吗？
  • 第二，延迟问题：
    识别，对话，合成，需要调用三次API。延迟是否很长？ 主要的延迟发生在什么位置？
  • 第三，界面问题：
    作为一个程序，需要一个界面，来便于操作。

1.2 制作：GUI及声音收录

ChatGPT用提示词：

写一个python程序来收集语音数据，需要的功能如下：
1. 制作一个UI界面：上面有一个开始按钮； 一个设备列表用来选择输入设备； 还要有一个区域，用来显示输入语音的能量（可以用波形来表示）
2. 将所有可选语音输入设备，初始化到UI界面中的列表中，操作系统为Windows10.
3. 语音收录条件如下：点击开始按钮后，根据能量值，自动检测所选的设备的输入是否有声音，如果检测到连续的0.3秒声音，开始录制声音，当出现0.5s没有声音时，自动停止录音
4. 收录停止时，将收录的声音存储成16k的wav文件。wav文件名，采用日期的格式，精确到秒级别。

ChatGPT写的程序可以运行，但是也出现了几个小问题
ChatGPT写的程序出现的几个问题

  1. 收录阻塞，录音无法停止
  2. 保存文件时没有进行验证
  3. 判断语音，非语音的能量值设置过大

使用ChatGPT调试代码的建议：

  • 将执行的问题反馈给它
  • 将错误直接复制给它
  • 设计到数据时，打印出数据的结构，把数据结构反馈给它

1.3 API调用部分

目标是什么？

  • 调用API，处理语音识别，问题解答，语音合成。

考虑到功能比较独立，分步来制作。首先实现语音识别的部分。

ChatGPT用提示词

新建一个文件 OpenAI_API.py，定义一个新的类，用来调用OpenAI API来处理语音识别，文本处理，语音合成。

先准备语音识别函数，功能如下：
输入为wav文件，输出为经过API处理后返回的文本。
文本要在GUI界面上显示。

API调用的代码，可以参考下面的，也可以参考OpenAI网站上的
from openai import OpenAI
client = OpenAI()

audio_file= open("/path/to/file/audio.mp3", "rb")
transcript = client.audio.transcriptions.create(
  model="whisper-1", 
  file=audio_file
)

主要出现的问题是，ChatGPT并不掌握API最新的信息。
需要我们自己来调试，或者打印出相关的结果给ChatGPT反馈。

调通了语音识别之后，再分别让它准备文本处理，和语音合成部分。

1.4 遗留的问题与挑战

  1. 讲话不连续的影响？讲话出错时重讲是否影响大模型理解？
  2. 控制大模型输出：大模型回答倾向于全面，对话时会比较啰嗦，如来来控制？ 或者如何来打断它？
  3. LLM可以支持Streaming的处理，是否可以结合语音合成来减少延迟时间？
  4. 界面的美化。
